Million Arthur Unofficial Desktop ClientOct 2013 ~ Present

• Implemented by using C# and .Net Framework.

• Encrypted server-client transmission.

• Object Serialization.

• SQLite database processing.

• Adopted Multithreading to handle numbers of HTTP requests.

<Click for More Information>

Million Arthur is a mobile card game developed by Square Enix, which is also the developer of Final Fantasy series, Dragon Quest series and many other famous games.

The unofficial desktop game client was developed in a team of two. It is implemented by using C# and .Net Framework in Microsoft Visual Studio. SQLite was used to save the data of game client as well as the current version of the game client. All HTTP requests and responses were encrypted and decrypted before sending and receiving. We adopted Multithreading to organize those requests and responses from different user accounts. Also, synchronized the information from the server with the UI of the game client.

We designed a user-friendly UI so that player can have the same experience as the official mobile client. Moreover, we provided some extra functionalities to ease the gameplay. The application won the approval of public. We released this application to public at the end of November 2013. And we had over 2,000 users within one month.

Picture Compression ApplicationMar 2012

• Developed an application by using C# to lossy compress pictures.

• Used basic image compression algorithms to implement the whole process.

<Click for More Information>

This application uses the basic JPEG compression workflow to perform the image compression from a .bmp file into .IM3 file which is defined by the requirement of this project.

Following are the compression steps:

• Block Preparation: Convert RGB color to YUV color. And separate YUV into three blocks with 4:2:2 Downsampling.

• Transform: Apply 2D Discrete Cosine Transform (DCT) on each block.

• Quantization: Quantized DCT Coefficients. The quantization table is choosed by the quality of compression user selected in the UI.

• Zigzag scan through each block.

• Differential Pulse Code Modulation (DPCM) on DC component.

• Run Length Encoding (RLE) on AC Components.

• Entropy Coding the lossy compressed file.

Raytrace Rendering ApplicationNov 2011

• Developed a 3D rendering application using C++ which output realistic rendered images.

• Used Raytrace as main rendering algorithm and applied matrix calculation to calculate vertices in coordinate.

• Wrote a file reader which can get data from ppm and scn files.

<Click for More Information>

This is a terminal application which can render 3D scene with Raytrace illuminate algorithm.

The application can read .scn file (This file defines the output file name, output image resolution, scene light sources, and objects in scene. For each objects, it contains the information of 3D object file to be used (.obj file), reflection and refraction ratio, position in the scene, transformation values, and the texture being used if any.). After reading the scene file, the application would render the output image pixel by pixel and create .ppm image once it finished.
